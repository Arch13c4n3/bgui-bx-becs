.BGUI - design language for BasicGUI 
'blazing fast GUI for Raylib games' (will soon port to vulkan backend but idk)

compilation step: syntax tree parser -> 
	c++ compiler & separate binary file for fast start ups 
	(the bin file will only be reparsed/updated if save changes are valid)
how? : the basicGUI.hpp library will find the file specified in the <windowInstance>.use_design_language("<file PATH>");

Soon features: real time syntax highlighting, animation handling , support multiple files

key features:
* simple , no redundancy, concise , type safe
* concise error handling - log errors at parsing time(during ui assembly) and run time(for hot reloading)
* HOT RELOAD - any changes will reflect immediately to the running program , if invalid the parser will log the error
* OOP inheritance - properties of a base component can be inherited by a child component but can be overridden
	by reassigning a value to a non-exclusive member already assigned
* themes - applied styling to components
* high level dynamic containers - for storing of elements / components, can have layout definition (list, grid, etc..).
	by default inherits the layout of the global defined.
	containers with layout can contain text
* can be visually edited using the 'BGUI editor'!

Syntax:
*macros: 
'@' for declaring components
'#' for declaring global styling
'{}' contains the body where properties are defined

*keywords:
'exclusive' - defines a property whose value cannot be inherited.


*initial styling: (optionals)
------------------------------
#theme default // uses raylib default text and bgui default style
#font_style default // uses raylib default text
#font_size <num> 
#background_color <color>
#layout default // uses grid layout style by default
------------------------------

*component declaration:
*normal
---------------------------
@<component type> <name> {
	<member> = <value>
	 ...
}
--------------------------------

*child component
------------------------------
// the parser will automatically infer the type of the base
@<base component name> <name> {
	...
	theme = <theme> // will override globals at this scope
	font_style = 
}
---------------------------
practical example:
---------------------------
@button button1 {
	
}
---------------------------
access in .cpp 
(access only when needed aka. adding custom events functionality)
---------------------------
#include "BGUI.hpp"

BGUI::game(){
 	auto MyGui = use_bdl("<path to (.bdl) file>"); // fetch bdl instance
	auto button1 = MyGui.use_component("button1"); // finds button 1 in specified bdl file and assigns instance to variable
	
// can now be used:
	button1.event.onClick = []{
		//custom onclick here
	};

	button1.properties = {.height = 500, .width= 400}; // <- override values 
}
